# 项目文档管理规范

> 本文档定义项目文档的组织结构、分类策略和维护规范。
> **最后更新**: 2026-02-18

## 文档目录结构

```
docs/
├── DEVELOPMENT_PLAN.md      # 开发计划总览
├── UNIMPLEMENTED_TASKS.md   # 未落地任务清单
├── README.md                # 项目说明文档
│
├── plans/                   # 设计方案文档
│   └── YYYY-MM-DD-*.md      # 按日期命名的设计方案
│
├── specs/                   # 功能规格文档
│   ├── auto-dev-server/     # Auto Dev Server 规格
│   ├── dark-mode-toggle/    # 深色模式切换规格
│   ├── litellm-integration/ # LiteLLM 集成规格
│   ├── remotion-video-generation/ # 视频生成规格
│   ├── video-transcription/ # 视频转录规格
│   └── mcp-publish-platform/ # 发布平台规格
│
├── dev/                     # 开发指南文档
│   ├── AI_DEVELOPER_GUIDELINES.md  # AI 开发者准则
│   ├── DEV_WORKFLOW.md             # 开发工作流
│   ├── SPEC_STANDARDS.md           # 规格文档标准
│   └── ISSUE_*.md                  # 问题修复记录
│
└── api/                     # API 文档
    └── *.md                 # API 接口说明
```

## 文档分类说明

### 1. 计划类文档 (Plans)

| 文档 | 用途 | 更新频率 |
|------|------|----------|
| DEVELOPMENT_PLAN.md | 项目整体开发计划和进度 | 每周或重大变更时 |
| UNIMPLEMENTED_TASKS.md | 未完成任务追踪 | 任务状态变更时 |
| plans/*.md | 具体功能设计方案 | 设计完成后归档 |

### 2. 规格类文档 (Specs)

每个功能模块包含以下文件：

| 文件 | 用途 |
|------|------|
| requirements.md | 功能需求描述 |
| design.md | 技术设计方案 |
| tasklist.md | 任务清单和进度 |

### 3. 指南类文档 (Dev)

| 文档 | 用途 | 目标读者 |
|------|------|----------|
| AI_DEVELOPER_GUIDELINES.md | AI 开发者行为准则 | AI 助手 |
| DEV_WORKFLOW.md | 开发工作流程 | 开发者 |
| SPEC_STANDARDS.md | 规格文档编写标准 | 开发者 |

### 4. API 文档 (API)

| 文档 | 用途 |
|------|------|
| API 接口说明 | 描述 API 端点、参数、响应 |

## 文档命名规范

### 文件命名

- 使用小写字母和连字符
- 版本化文档使用日期前缀：`YYYY-MM-DD-feature-name.md`
- 问题记录使用前缀：`ISSUE_description.md`

### 标题规范

- 一级标题：文档名称
- 二级标题：主要章节
- 三级标题：子章节

## 文档维护规范

### 更新时机

1. **功能完成时**：更新任务状态、完成日期
2. **需求变更时**：更新需求文档、设计方案
3. **问题修复时**：记录问题原因和解决方案
4. **定期审查**：每周检查文档一致性

### 状态标记

| 标记 | 含义 |
|------|------|
| ✅ | 已完成 |
| 🔄 | 进行中 |
| ⏳ | 待开始 |
| ⚠️ | 部分完成/有风险 |
| ❌ | 已取消 |

### 版本控制

- 所有文档变更需提交到 Git
- 提交信息格式：`docs: 更新内容描述`
- 重大变更需在文档底部添加更新记录

## 文档质量检查清单

### 新建文档检查

- [ ] 文件命名符合规范
- [ ] 包含创建日期和最后更新日期
- [ ] 目录结构正确
- [ ] 内容格式统一

### 更新文档检查

- [ ] 更新最后更新日期
- [ ] 状态标记正确
- [ ] 交叉引用有效
- [ ] 无过时信息

## 文档审查周期

| 文档类型 | 审查周期 |
|----------|----------|
| 开发计划 | 每周 |
| 任务清单 | 每周 |
| 规格文档 | 每月 |
| 指南文档 | 每季度 |
